PHP Scripting Framework

A lightweight, modern PHP 8 micro-framework with routing, ORM, migrations, template engine, DI container, and an artisan-like CLI — zero bloat, full control.

PHP 8.1+ MySQL / PDO Composer PSR-4 Plates Templates Pimple DI

Overview

This framework is built to give you the ergonomics of a full-stack framework (routing, ORM, CLI, validation, template inheritance) without the weight. Every component is replaceable and the codebase stays small enough to read in an afternoon.

Setup

# 1. Clone and install dependencies
git clone https://github.com/yourname/scripting.git
cd scripting
composer install

# 2. Create your database
mysql -u root -p -e "CREATE DATABASE scripting CHARACTER SET utf8mb4"

# 3. Configure credentials in config/db.php (see Configuration section)

# 4. Run migrations
php bin/artisan migrate

# 5. Point your web server's document root to /public
# Apache: set DocumentRoot to /public
# Or just serve via XAMPP and browse http://localhost/scripting/public

Configuration

    'host'     => 'localhost',
    'dbname'   => 'scripting',
    'username' => 'admin',
    'password' => 'secret',
];

App — config/app.php

return [
    'app_name'    => 'My App',
    'base_url'    => '/scripting/public',  // no trailing slash
    'timezone'    => 'Asia/Dhaka',
    'app_mode'    => 'development',         // or 'production'
    'view_folder' => __DIR__ . '/../htmls/views',

Note Config files are loaded automatically from config/*.php — any file you add there becomes accessible via app()->getConfig('filename').

Routing

Routes are declared directly on controller methods using PHP 8 attributes — no separate route file needed.

use Src\Attributes\Route;

class PostController extends Controller
{
    // Basic route
    #[Route('GET', '/posts')]
    public function index() { ... }

    // Named route with dynamic parameter
    #[Route('GET', '/posts/{id}', 'post.show')]
    public function show($id) { ... }

    // POST route — CSRF middleware applied automatically
    #[Route('POST', '/posts', 'post.store')]
    public function store() { ... }
}

Named Routes

Generate URLs from named routes using the router()->route() helper:

// In a view:
$url = router()->route('post.show', ['id' => 42]);
// → /scripting/public/posts/42

Route Attribute Signature

ArgumentTypeRequiredDescription
methodstringYes'GET' or 'POST'
pathstringYesURL path, e.g. '/users/{id}'
namestringNoNamed route alias for URL generation
CSRF Protection Every POST route automatically runs CsrfMiddleware. Include the hidden token in every form:
<input type="hidden" name="csrf_token" value="<?= generateCsrfToken() ?>">

Controllers

Controllers live in src/Controllers/ and extend Controller. They are auto-discovered — just create the file.

namespace Src\Controllers;

use Src\Attributes\Route;

class ArticleController extends Controller
{
    #[Route('GET', '/articles')]
    public function index()
    {
        $articles = $this->db()->from('articles')->latest()->paginate(10);
        $this->render('articles/index', compact('articles'));
    }
}

Base Controller Methods

$this->render(string $view, array $data)Render a Plates template with data
$this->json(mixed $data, int $code = 200)Output JSON response and exit
$this->db(): DatabaseGet the Database singleton
$this->post(string $key, ?string $rules)Get POST field (with optional validation rule)
$this->get(string $key)Get query string value
$this->validation(): arrayReturns validation errors array
$this->renderErrors(): stringFormats validation errors as HTML string

Scaffold a Controller

# Basic controller
php bin/artisan make:controller ArticleController

# Resource controller (index, store, show, update, delete)
php bin/artisan make:controller ArticleController --resource

#[Guest]
#[Route('GET', '/login')]

Middleware

Middleware classes live in src/Middleware/. The built-in CsrfMiddleware is applied automatically to all POST routes.

# Generate a middleware stub
php bin/artisan make:middleware RateLimitMiddleware
namespace Src\Middleware;

class RateLimitMiddleware
{
    public function handle(): void
    {
        // Inspect request, set headers, or abort
    }
}

Attach middleware to routes by passing it as the 4th argument to the router's add(), or register it inside a route group.

Query Builder

Src\Database is a fluent query builder backed by PDO. Get the instance anywhere via Database::getInstance() or $this->db() inside a controller.

Reading

// Select all
$rows = $db->from('users')->get();

// Filtering & ordering
$user = $db->from('users')
           ->where('email', '=', 'alice@example.com')
           ->first();

// Full-text LIKE search
$results = $db->from('users')
              ->whereLike('first_name', 'ali')   // wraps with %
              ->latest('created_at')
              ->paginate(15);

Paginate Response

$result = $db->from('posts')->latest()->paginate(10);

// $result is a stdClass:
$result->total;    // total record count
$result->pages;    // total page count
$result->per_page; // items per page
$result->data;     // array of stdClass rows
$result->links;    // pre-rendered pagination HTML

Writing

// Insert
$db->insert('posts', [
    'title'   => $title,
    'body'    => $body,
    'user_id' => 1,
]);

// Update
$db->update('posts',
    ['title' => 'New Title'],      // SET
    ['id'    => 42]               // WHERE
);

// Delete
$db->delete('posts', ['id' => 42]);

Raw PDO

$pdo = Database::getInstance()->getConnection();
$stmt = $pdo->prepare("SELECT * FROM users WHERE role = ?");
$stmt->execute(['admin']);
$admins = $stmt->fetchAll();

Models

Models in src/Models/ extend Model. The table name is auto-derived (class name lowercased + "s") but can be overridden.

# Scaffold a model (optionally with its migration)
php bin/artisan make:model Post
php bin/artisan make:model Post --migration   # -m also works
namespace Src\Models;

class Post extends Model
{
    protected $table    = 'posts';
    protected $fillable = ['title', 'body', 'user_id'];
    protected $hidden   = [];

    // Custom methods are fine
    public function isPublished(): bool
    {
        return !empty($this->published_at);
    }
}

Migrations

Migrations live in database/migrations/ as timestamped PHP classes. The framework tracks which have run in a migrations table.

Create a Migration

php bin/artisan make:migration create_posts_table

This generates database/migrations/2024_06_21_153000_create_posts_table.php:

class CreatePostsTable
{
    public function up()
    {
        $pdo = \Src\Database::getInstance()->getConnection();
        $pdo->exec("
            CREATE TABLE posts (
                id         INT AUTO_INCREMENT PRIMARY KEY,
                user_id    INT NOT NULL,
                title      VARCHAR(255) NOT NULL,
                body       TEXT,
                created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
                FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
            ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
        ");
    }

    public function down()
    {
        $pdo = \Src\Database::getInstance()->getConnection();
        $pdo->exec("DROP TABLE IF EXISTS posts");
    }
}

Migration Commands

  • php bin/artisan migrate Run all pending migrations
  • php bin/artisan migrate:status Show ran / pending status of every migration
  • php bin/artisan migrate:rollback Roll back the last batch of migrations
  • php bin/artisan migrate:refresh Roll back ALL then re-run — ⚠️ clears data
Best Practices One logical change per migration file. Always write down(). Use descriptive snake_case names like add_avatar_to_users.

Views & Templating

Views use Plates — a native-PHP template engine. View files live in the folder configured at config/app.php → view_folder.

Layouts

Custom Commands

Create a class in src/Console/Commands/ that extends Command, then register it in Kernel.php.

namespace Src\Console\Commands;

use Src\Console\Command;

class SendNewsletterCommand extends Command
{
    protected string $name        = 'newsletter:send';
    protected string $description = 'Send the weekly newsletter';

    public function handle(array $args): int
    {
        $dry = in_array('--dry-run', $args);

        $this->info("Sending newsletter...");

        $users = $this->db->from('users')->get();

        foreach ($users as $user) {
            if (!$dry) {
                // mail($user->email, ...)
            }
            $this->line("  → {$user->email}");
        }

        $this->success(count($users) . " emails dispatched");
        return 0; // 0 = success, 1 = failure
    }
}

Register in src/Console/Kernel.php → registerCommands():

$this->register(new SendNewsletterCommand());

Command Output Methods

$this->info(string)ℹ️ Informational message
$this->success(string)✅ Success message
$this->error(string)❌ Error message
$this->warn(string)⚠️ Warning message
$this->line(string)Plain output line
$this->table(headers, rows)Formatted ASCII table
Database Access in Commands Use $this->db — it is the Database singleton pre-injected in the base Command constructor.

Validation

Validation rules are declared inline when reading POST fields:

$name  = $this->post('name',  'required|min:3|max:100');
$email = $this->post('email', 'required|email');
$age   = $this->post('age',   'required|numeric');

$errors = $this->validation();

if ($errors) {
    flash('error', $this->renderErrors());
    redirect('/form');
}

Available Rules

RuleDescription
requiredField must not be empty
min:NMinimum string length of N
max:NMaximum string length of N
emailMust be a valid email address
numericMust be numeric

Dependency Injection

The container is a Pimple instance, bootstrapped in Application::registerServices(). Bind your own services there or in a Service Provider.

// Application::registerServices()
$this->container['mailer'] = function() {
    return new Mailer(config('mail'));
};

// Resolve anywhere
$mailer = app()->getContainer()['mailer'];

Logging

Powered by Monolog. A default request_logger writes to logs/request_time.log.

// Inject from container or use directly
$log = app()->getContainer()['logger'];
$log->info('User logged in', ['user_id' => $id]);
$log->error('Payment failed', ['order' => $orderId]);

Error Handling

ErrorHandler::handleException() is registered globally via set_exception_handler(). It automatically detects CLI vs HTTP context:

ContextBehaviour
HTTP — developmentRenders views/errors/error.php with the full message
HTTP — productionRenders generic views/errors/500.php (hides details)
HTTP — 404Renders views/errors/404.php
CLIPrints ❌ Error: ... to stdout and exits with code 1

Throwing Application Exceptions

use Src\Exceptions\AppException;

throw new AppException('Not found', 404);
throw new AppException('Forbidden', 403);

Service Providers

Any class in src/Providers/ that extends ServiceProvider and implements register(Container $c) is auto-loaded at boot time.

namespace Src\Providers;

use Pimple\Container;

class MailServiceProvider extends ServiceProvider
{
    public function register(Container $c): void
    {
        $c['mailer'] = fn() => new Mailer(app()->getConfig('mail'));
    }
}

PHP Attributes

The framework leverages PHP 8 attributes for routing and access control — no annotation parsing, no magic strings.

AttributeLocationEffect
#[Route(method, path, name?)]Controller methodRegisters the method as an HTTP route
#[Auth]Controller methodRedirects to /login if no session
#[Guest]Controller methodRedirects to / if already logged in
#[Auth]
#[Route('GET', '/settings')]
public function settings()
{
    // Only reachable when logged in
}
Multiple attributes on one method Stacking attributes is fully supported — list them in any order above the method signature.
